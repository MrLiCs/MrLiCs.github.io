SSM案例之项目结构搭建

发表于 2019-08-02

由于对IntelliJ IDEA不熟悉，故将编程工具更换为 Eclipse

使用Maven管理该项目,版本为maven3优点：统一版本管理、工程间依赖和Jar包管理方便、可自动打包等

需要配置Maven的本地仓库，配置方法可以百度查询。因为需要的包不多，所以使用本地仓库。

工程结构

Taotao-parent：父工程，打包方式pom，统一管理jar包的版本号。

​ |项目中所有工程都应该继承父工程。

​ |–Taotao-common：通用的工具类通用的pojo,util。打包方式jar

​ |–Taotao-manager：服务层工程，聚合工程，Pom工程

​ |–taotao-manager-dao：打包方式jar

​ |–taotao-manager-pojo：打包方式jar

​ |–taotao-manager-interface：打包方式jar

|–taotao-manager-service：打包方式：war (为了发布服务的方便)

|–taotao-manager-web：表现层工程，例如jsp页面，打包方式war

###根据工程结构建立maven工程，所有工程都继承Taotao-parent工程。Taotao-manager工程下，分别有三个子工程

pojo用于存放Beans所以不需要依赖其他；

dao需要与数据库交互，所以需要依赖pojo，所以要到dao中的pom.xml配置pojo依赖；

interface需要返回数据，所以也需要依赖pojo，也需要在pom.xml配置pojo依赖；

service需要调用dao于interface，所以需要配置dao和interface到pom.xml中，由于需要返回数据，所以也需要配置pojo依赖，但由于dao和interf中都有配置pojo依赖，通过传递性，service也可以使用pojo

启动taotao-manager工程

需要在taotao-manager工程下pom.xml中配置tomcat插件：

配置完成后，记得在RunAs—>Maven install一下！！！！这样才不会报错！因为包需要进行install！！！

依次Run As—>Maven build—>弹出窗口，在窗口中找到Goals，并输入 clean tomcat7:run；注：tomcat6可以直接写tomcat：run

####taotao-manager-web也需在pom.xml中配置tomcat插件，但省去Maven install，直接Run As—>Maven build—>弹出窗口，在窗口中找到Goals，并输入 clean tomcat7:run
